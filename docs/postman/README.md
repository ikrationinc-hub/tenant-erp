# Postman collection

`hyperion-erp.postman_collection.json` + `hyperion-erp.postman_environment.json` cover every route currently mounted in `apps/api/src/app.ts`, generated by reading the actual `*.routes.ts` and `*.validator.ts` files (not hand-typed), so bodies match what each endpoint's Zod schema actually accepts.

## Import

1. Postman → Import → select both files.
2. Select the **Hyperion ERP - Local** environment (top-right dropdown).
3. Run **Auth > Login** (tenant) and/or **Platform Admin / Auth > Login** (platform admin) once — their Tests scripts auto-capture `accessToken`/`refreshToken` (or the platform equivalents) into environment variables. Every other request in the collection already points at `{{accessToken}}` / `{{platformAccessToken}}`, so nothing else needs manual copy-pasting.

## Folders

| Folder | Auth |
|---|---|
| Health | none |
| Platform Admin / Auth, Platform Admin / Tenants | platform-admin bearer token |
| Auth, Users & Invitations, Menus, Field Definitions, Masters, Attachments, Suppliers, Purchase | tenant bearer token |

Platform-admin and tenant tokens are signed with **different secrets** — a platform token 401s on tenant routes and vice versa. Don't swap `platformAccessToken` and `accessToken`.

## Things baked into the requests, not just left as tribal knowledge

- **`tenantCode`**: required on `Auth > Login` and the invitation endpoints when there's no subdomain to resolve the tenant from — which on localhost is always. Set it in the environment once per tenant you're testing against.
- **Decimal fields are strings**: `Purchase > Add purchase item`'s `quantity`/`purchaseRateUsd`/`exchangeRate` are quoted decimal strings (`"25.500000"`), never JSON numbers — that's what the API actually accepts (`decimalStringSchema` in `purchase-items.validator.ts`), and it's also CLAUDE.md rule 1.
- **Masters**: one generic CRUD pattern (`core/masters/factory.ts`) instantiated 15 times. Only **Countries** carries the full inline explanation; the other 14 subfolders (Cities, Currencies, Payment Terms, UOM, Ports, Warehouses, Incoterms, Items, Item Grades, Vessels, Transport Modes, LME Exchanges, Hedge Platforms, Supplier Types) are byte-for-byte the same 7-endpoint shape with a different URL segment.
- **Strict bodies**: most POST/PATCH bodies are `.strict()` Zod schemas — sending a field the endpoint doesn't recognize is a 422, not a silently-dropped no-op. Don't add fields "just in case" when testing.
- **`GET /users/me/companies`** is deliberately **not** in this collection — it doesn't exist on the backend yet (see conversation history / `packages/contracts/src/scope.ts`'s comment). Nothing to test against until it's built.

## Regenerating

This collection was produced by a small Node script (reads route/validator files, emits Postman v2.1 JSON) rather than maintained by hand, so it doesn't silently drift as much as a hand-edited export would — but it is still a snapshot as of 2026-07-23, not a live introspection of the running server. If you want the generator script itself checked into the repo (e.g. under `scripts/`) so it can be re-run after adding new endpoints, ask and it'll be added — it wasn't included here to avoid speculative tooling nobody asked for yet.
